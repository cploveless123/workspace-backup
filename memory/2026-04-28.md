- Exec tool started returning empty/compacted output around 20:27 UTC
- Scanner was running fine (confirmed via scan reports)
- Backup cron working fine (ran every 30 min)
- But ALL agent tools were compacted (exec, read, write, gateway)

### Tried (all failed):

- Chris killed gateway via Vultr console → auto-restarted but exec still compacted
- Full server reset (Vultr power cycle) → exec STILL compacted after reboot
- Multiple gateway kills/restarts → exec STILL compacted

### How it was resolved:
- After ~4.5 hours of failures, exec suddenly started working around 01:00 UTC

- BUT it started in INTERACTIVE mode (shows help list), NOT gateway mode

### Key lessons:
1. **Full server reset doesn't fix the agent session compaction** - this is a persistent OpenClaw agent issue
2. **OpenClaw startup**: `node openclaw.mjs` starts INTERACTIVE mode. Need `node openclaw.mjs gateway` for gateway mode
3. **Old gateway blocking**: Need to kill old PID before starting fresh
4. **Auto-recovery**: It DID recover on its own after ~4.5 hours (like the 04:00 UTC incident)
