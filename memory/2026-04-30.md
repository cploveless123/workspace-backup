## Kitcat Buy Post-Mortem — Root Cause Found

**What happened:** Token bought at $2,347 mcap (well below $8K minimum), stopped out in 1 second (-0.074 SOL).

**Root cause — `get_fresh_token_data()` broken for brand new tokens:**

1. GMGN kline → `None` (token too new, no candles yet)
2. GMGN token info fallback condition → `info.get('price_change_percent1h') is not None` → **ALWAYS FALSE** (GMGN never provides h1 in token info — only in kline)
3. DexScreener fallback condition → `pairs[0].get('priceChange', {}).get('h1') is not None` → **ALWAYS FALSE** for pump.fun tokens (DexScreener doesn't track h1 for pump)

**Result:** All 3 paths return `None, None` → `fresh_mcap = 0` → fresh mcap check was skipped → bad buy proceeded.

**Fix applied (2026-04-30 22:30 UTC):**
- Added block: `if fresh_mcap == 0: return False, "Cannot verify mcap"` in `buy_token`
- Scanner committed with fix

**Additional fix needed:**
