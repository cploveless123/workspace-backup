### H1 Scoring Discovery — CRITICAL
- signal_scorer.py was using INVERTED H1 logic based on GMGN general data
- GMGN data (1623 trades): higher H1 = better WR (general population)
- OUR trades (pump path only): lower H1 = better WR (101-150% = 52% WR, >400% = 0% WR)
- These two populations are DIFFERENT — pump path is pre-filtered
- Root cause: I cited GMGN data but applied it to our already-filtered pump path subset
- **Fixed:** signal_scorer H1 scoring now inverts — lower H1 = more points
  - 101-150%: +2.0 pts (52% WR)
  - 151-200%: +1.5 pts
  - 201-300%: +1.0 pts
  - 301-400%: +0.5 pts
  - >400%: 0 pts
