
## 2026-05-04 Trading Bot Rewrite

### Major Issues Found & Fixed

1. **GMGN API Field Changes** - CRITICAL
   - GMGN changed API response structure without notice
   - Old fields: `market_cap`, `h1`, `volume_24h`, `exchange` → all returning 0
   - New fields: `price`, `supply`, `stat` object, `launchpad`
   - Fixed scanner to use correct fields: mcap = `price * supply`, stats from `stat` object
   - **Lesson:** Always verify API responses, never assume field names stay the same

2. **Old System Alerts Still Sending**
   - alert_sender.py (old system) was still running and sending outdated reports
   - Old reports showed wrong data (balance -13 SOL, old positions, wrong settings)
   - Killed old alert_sender, now only v1.0 alerts
   - **Lesson:** When deploying new system, kill ALL old processes

3. **Multiple Scanner/Monitor Versions**
   - Had gmgn_scanner.py, position_monitor.py, scanner_simple.py, monitor_simple.py
   - All running simultaneously causing confusion
   - Archived old versions, only running scanner_v1.py + monitor_v1.py
   - **Lesson:** Clean up old files before deploying new versions

4. **Monitor Mcap Calculations Wrong**
   - Was using `entry_mcap * (current_price / entry_price)` for TP mcap
   - This gives WRONG results when supply changes
   - Fixed to use GMGN data directly: `price * supply`
   - **Lesson:** Never calculate what API provides directly

## 2026-05-04 23:13 UTC - PnL Calculation Fixes

### Problem: "This Trade" showing 0.0000 SOL
- TP1 trail stop was showing 0.0000 SOL even when position was at -58% loss
- Root cause: get_trade_earned() returned 0 because no sells happened for TP1 (just trail exit)
- Fixed: TP1 trail now calculates PnL directly: `(current_price - entry_price) * position_size / entry_price`

### Problem: "Total Earned" showing -15 SOL (impossible)
- Was accumulating all old trades ever made
- User clarified: "Total Earned" should be total for THIS TOKEN only, not cumulative
- Fixed: Changed all alerts to use trade_earned (sum of sells for this token) for both "This Trade" and "Total Earned"
- Now both show the net PnL for that specific token

### Problem: Old trades had corrupted pnl_sol values
- Old sell records didn't have pnl_sol field or had wrong values
- Fixed get_trade_earned() to calculate from entry_price and sell_price when pnl_sol is missing
- Works for both old trades (no pnl_sol) and new trades (with pnl_sol)

### Current Alert Format:
- This Trade: X.XXXX SOL (net for this token)
- Total Earned: X.XXXX SOL (same as This Trade - total for this token)
- Balance: X.XXXX SOL (wallet balance)

### Example for Animalcoin:
- TP2: +0.0429 SOL (sold 40% at profit)
- TP2_TRAIL: -0.0454 SOL (lost remaining 60%)
- Total Earned: -0.0025 SOL (net for this token)
