
## 2026-05-04 Trading Bot Rewrite

### Major Issues Found & Fixed

1. **GMGN API Field Changes** - CRITICAL
   - GMGN changed API response structure without notice
   - Old fields: `market_cap`, `h1`, `volume_24h`, `exchange` → all returning 0
   - New fields: `price`, `supply`, `stat` object, `launchpad`
   - Fixed scanner to use correct fields: mcap = `price * supply`, stats from `stat` object
   - **Lesson:** Always verify API responses, never assume field names stay the same

2. **Old System Alerts Still Sending**
   - alert_sender.py (old system) was still running and sending outdated reports
   - Old reports showed wrong data (balance -13 SOL, old positions, wrong settings)
   - Killed old alert_sender, now only v1.0 alerts
   - **Lesson:** When deploying new system, kill ALL old processes

3. **Multiple Scanner/Monitor Versions**
   - Had gmgn_scanner.py, position_monitor.py, scanner_simple.py, monitor_simple.py
   - All running simultaneously causing confusion
   - Archived old versions, only running scanner_v1.py + monitor_v1.py
   - **Lesson:** Clean up old files before deploying new versions

4. **Monitor Mcap Calculations Wrong**
   - Was using `entry_mcap * (current_price / entry_price)` for TP mcap
   - This gives WRONG results when supply changes
   - Fixed to use GMGN data directly: `price * supply`
   - **Lesson:** Never calculate what API provides directly

### Key Lessons
1. **Proactive monitoring** - When API returns $0, investigate immediately
2. **API field verification** - Check response structure regularly
3. **Clean deployments** - Kill old processes, archive old files
4. **Use API data directly** - Don't calculate what API provides
5. **Test before deploy** - Always verify changes work
