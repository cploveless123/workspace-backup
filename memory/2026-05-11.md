## Scanner Verification System Changes

### Changes Made (2026-05-10 ~23:00-23:45 UTC)

**1. Verification Stages Changed to 12+12+12**
- All paths (PHOENIX, DIP, WHALE, SNIPER_HEAVY, SNIPER) now have 3-stage verification: 12s + 12s + 12s = 36s total
- Previously was 10+10 (20s) after earlier change, now standardized to 12+12+12
- KOL, SMART_DEGEN, PROVEN paths have NO verification (buy immediately)

**2. DIP Path Criteria Widened**
- Dip range: -5% to -15% → -5% to -20% (catches more dips)
- chg1 requirement: > 0% → > -2% (allows slight negative while recovering)

**3. GMGN Failure Alerts Added**
- Added Telegram alerts when GMGN sources return 0 tokens or API errors
- Alerts for: trending 1m/5m/1h, trenches pump/new

**4. Critical Bug Fixed: `time` Variable Shadowing**
- Problem: Nested `import time` statements inside `if` blocks in `recovery_check()` function caused "local variable 'time' referenced before assignment" error
- This happened because Python marks `time` as a local variable for the entire function when it sees `import time` anywhere in the function, even inside `if` blocks
- Fix: Removed ALL nested `import time` statements since `time` is already imported at module level
- This was causing DIP and PHOENIX paths to crash during verification

**5. Watchdog Fixed**
- Scanner was being killed every 5 minutes by watchdog because log was "stale"
- Changed watchdog stale log check from 300s to 600s (10 minutes)
- Scanner now has time to complete multiple cycles without being killed
