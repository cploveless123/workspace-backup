## Data Recording Bug Discovered & Fixed (23:20-23:31 UTC)

**Issue:** Scanner logged 66 BOUGHT signals since 07:45 UTC, but trades file only had 12 buys recorded. 54 buys were missing.

**Root Cause:** Race condition between scanner and monitor:
- `monitor_v1.py` used `fcntl.LOCK_EX` in `close_position()` (read-modify-write pattern)
- `scanner_v1.py` did NOT use file locking in `buy_token()` (simple append)
- Result: Monitor would read file, scanner would append, monitor would truncate and write back → scanner's buy LOST

**Fix Applied:**
- Added `import fcntl` to scanner_v1.py (after datetime import)
- Wrapped file write in `buy_token()` with exclusive lock:
  ```python
  with open(TRADES_FILE, 'a') as f:
      fcntl.flock(f, fcntl.LOCK_EX)
      f.write(json.dumps(trade) + '\n')
      f.flush()
      fcntl.flock(f, fcntl.LOCK_UN)
  ```
- Also added locking to `record_sell()` in monitor_v1.py for consistency

**Safeguards Followed:**
- Created backups: scanner_v1.py.2026-05-12-232028 and monitor_v1.py.2026-05-12-232028
- Tested both modules import successfully
- Tested file locking with test trade (verified in file, then removed)
- Verified file integrity: 628 valid records
- Got Chris's explicit approval before deploying

**Restart:** Scanner and monitor restarted at 23:31 UTC with fix applied. Both running healthy.

**Impact:** All future trades should be recorded correctly. Before fix: ~18% capture rate. After fix: 100% (expected).
