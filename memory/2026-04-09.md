## CRITICAL BUG: closed_at vs fully_exited (02:XX UTC)
### The Bug
position_monitor.py skipped tokens with `closed_at` set, but `closed_at` is set for BOTH:
- Full exits (stop loss, rug, manual sell)
- TP1 partial exits (partial_exit = True, position still open)

This meant partial positions (LAPD, LOY-002 x2, Cupsey) were NEVER checked by the monitor after TP1 fired. Their trailing stops and stop losses were never triggered.

### Root Cause
The skip logic was: `if t.get('closed_at'): continue`
But TP1 partial exits ALSO set `closed_at` = timestamp of TP1 sale.

### The Fix
Introduced `fully_exited` flag:
- Set `fully_exited = True` on: stop loss, TP2, TP3, trailing stop (full exits)
- NOT set on: TP1 (partial exit - position still open)
- Skip logic: `if t.get('fully_exited'): continue`
- Partial exits (tp1_sold, not trailing_stopped) are still checked for TP2/TP3/stop

### Lesson
NEVER use `closed_at` alone to determine if a position is fully closed. Use a dedicated flag.
When setting `closed_at` for partial exits (TP1), ensure position is still being monitored.
The `status='open_partial'` with `tp1_sold=True` means position still needs monitoring.
Only `trailing_stopped=True` or explicit `fully_exited=True` means fully done.

### Files Fixed
- position_monitor.py: skip logic, fully_exited flag on full exits
- reports/current_trading_report.py, last5_trades.py, wilson_progress.py: use fully_exited check
